String Generation: The "Last Available Exit"

❓ What is the question asking?

You are given a rule string str1 (consisting of 'T' and 'F') of length N, and a target string str2 of length M. Your goal is to construct the lexicographically smallest (alphabetically first) string of length N + M - 1 that satisfies these rules:

T

If str1[i] == 'T', the substring starting at index i of length M MUST exactly match str2.

F

If str1[i] == 'F', the substring starting at index i of length M MUST NOT match str2.

Visual Example: str1 = "TF", str2 = "ab"

🧠 The Thinking Process & Intuition

1

Lock the Dictators

The 'T' rules are non-negotiable. Before guessing anything, we must place str2 exactly where every 'T' demands it. If two 'T' rules overlap and require different characters at the same spot, the task is impossible.

Greedy Start: Lock 'T's
2

Greedy Defaults

We want the smallest possible string. Therefore, for every remaining empty spot, our default choice should always be 'a'. We only change our mind if placing an 'a' breaks a rule.

The "Last Exit" Trapdoor

An 'F' rule only fails if the entire window matches str2. If we stand at the last empty spot of an 'F' window, and all previous spots match, we are forced to pick the next smallest valid character.
