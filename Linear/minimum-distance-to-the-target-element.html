Understanding the Problem

What is the question asking?

You are placed at a specific starting position (start) inside an array (nums). Your goal is to find a specific number (target). Because there might be multiple occurrences of this number, you want to find the one that requires the fewest steps to reach from your starting position.

Distance is calculated as the absolute difference between your current index and the target's index: abs(i - start).

Example Breakdown

Array: [1, 2, 3, 4, 5] | Target: 5 | Start: 3
- You start at index 3 (value 4).
- The target (5) is at index 4.
- Distance = abs(4 - 3) = 1.

The Thought Process

  1. Identify constraints: The array length is small (up to 1000). This means an $O(N)$ linear scan is perfectly acceptable and will execute in milliseconds.
  2. Formulate the brute-force plan: Iterate through the entire array. Every time I see a number that matches the target, I'll calculate how far it is from start.
  3. Track the minimum: Keep a running variable (like min_dist) initialized to infinity. Update it whenever a smaller valid distance is found.

Alternative "Clever" Approach

While looping from 0 to n-1 works great, there is a more optimal way to think about it logically: Expand Outwards.

If we start exactly at the start index and look left and right simultaneously (distance 0, then distance 1, then distance 2...), the very first time we find the target, it is guaranteed to be the closest one! We can stop immediately.

Pattern Recognition 🧩

This problem falls into two common patterns:

  • Linear Scan with State Tracking: A fundamental pattern where you walk through a collection and continually update an "answer" variable (like min, max, or sum) based on conditions.
  • Two-Pointer Expansion (Advanced thought): A pattern used when searching for the "closest" item. By starting at the origin and expanding outward with two pointers (`left` and `right`), you guarantee optimal early exits.
