DP Deep Dive: Min Path Sum

Keyboard Shortcuts: ← → Step, Space Play/Pause, M Toggle Mode, R Random Grid, T Theory

Problem Pattern & Strategy Breakdown

The Problem & Pattern

The Prompt: Given an m x n grid filled with non-negative numbers, find a path from top-left to bottom-right that minimizes the sum of all numbers along the path.

The Rule: You can only move DOWN or RIGHT.

Pattern: 2D Dynamic Programming. This problem asks for an "optimal" value (minimum), and the best path to a cell relies entirely on the best paths to its neighbors (Overlapping Subproblems + Optimal Substructure).

Ways to Solve
  • ✗
    Brute Force (DFS): Explore every possible path. Time complexity is O(2^(m+n)). Horribly slow for large grids.
  • ●
    Top-Down (Memoization): The purple visualizer. Recursively dive to the end, but save answers in a Dictionary. Time: O(m×n).
  • ✓
    Bottom-Up (Tabulation): The blue visualizer. Start at (0,0) and systematically calculate the cheapest route to every cell iteratively. Time: O(m×n).
Which is Best?

Both DP approaches have O(m×n) time and space complexity, but Bottom-Up (Tabulation) is the industry standard winner.

Why? No Call Stack. Top-Down uses recursion. On a massive 2000x2000 grid, the giant chain of recursive calls will trigger a StackOverflowException before it ever reaches the base case. Bottom-Up uses standard loops and avoids call stack overhead entirely!
